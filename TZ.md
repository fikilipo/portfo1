# 📐 Проект: «Сетка и Ритм» — интерактивный инструмент

> Уточнённая версия ТЗ. Оригинал сохранён в файле `TZ` (без изменений).
> Цель уточнения — устранить двусмысленности (алгоритмы тепловой карты и индикатора ритма, поведение ползунков, адаптивность, доступность), зафиксировать структуру и критерии готовности.

---

## 1. Концепция

Интерактивный веб‑инструмент, который показывает, как модульная сетка и типографический ритм влияют на восприятие контента. Пользователь переключает пресеты, крутит ползунки и в реальном времени видит:

- как элементы «садятся» на выбранную базовую сетку;
- куда в первую очередь падает взгляд (F/Z‑паттерн, тепловая карта);
- соответствует ли текущая вёрстка ритму (индикатор + пояснение);
- чем «правильная» вёрстка отличается от хаотичной (режим До/После);
- как страница выглядит для дальтоников.

Сайт — одностраничный, полностью клиентский, разворачивается как статика.

---

## 2. Стек и поставка

Выбран **Вариант A** (по оригинальному ТЗ):

- Чистый HTML / CSS / JS, без сборки.
- CSS Custom Properties для всех параметров дизайна (`--grid-base`, `--line-height`, `--heading-size`, …).
- CSS Grid + Flexbox для раскладки.
- Vanilla JS (ES modules) для логики.
- Canvas 2D API — тепловая карта.
- Внешние зависимости подгружаются с CDN и опциональны:
  - `html2canvas` — экспорт превью в PNG;
  - ничего больше.

Причина выбора A: статический сайт легко деплоится на любой CDN/статик‑хостинг (в т.ч. `devinapps.com`), нет build‑шага, исходники читаемы «как есть».

## 4. Интерфейс

### 4.1 Шапка

- Название проекта «Сетка и Ритм».
- Переключатель пресетов: `Лендинг` · `Статья` · `Карточка товара` (сегментированный контрол).
- Кнопки: `Сохранить PNG`, `Режим «До/После»`, `Звук`.

### 4.2 Левая панель — «Параметры»

- **Базовый модуль сетки**: `4 / 6 / 8 / 12 px` (radio‑кнопки).
- **Ползунки**:
  - Размер заголовка `16–72 px` (шаг 1).
  - Межстрочный интервал `1.0–2.0` (шаг 0.05).
  - Отступы между секциями `16–128 px` (шаг 4).
  - Ширина контейнера `320–1200 px` (шаг 8).
- **Слои** (чекбоксы):
  - Сетка (вкл. по умолчанию).
  - Тепловая карта.
  - Привязка к сетке (snap) — при включении ползунки щёлкают по кратным значениям модуля.
- **Дальтонизм**: `Нет / Протанопия / Дейтеранопия / Тританопия`.

### 4.3 Центральная сцена

- Контейнер с выбранным пресетом. Ширина управляется ползунком.
- Поверх — `SVG`‑слой сетки и `Canvas`‑слой тепловой карты. Оба pointer‑events: none.
- При клике на значимый элемент (заголовок, CTA, цитата и т.п.) справа появляется **«заметка дизайнера»** — 1‑2 предложения, почему этот элемент так свёрстан (например: «CTA отделён от фичей отступом 3×модуль, чтобы создать визуальный разрыв»).

### 4.4 Правая панель — «Ритм»

- Большой бейдж: 🟢 «Ритм соблюдён» / 🟡 «Есть отклонения» / 🔴 «Ритм нарушен».
- Список конкретных замечаний: какие величины не кратны модулю, где межстрочный интервал «ломает» baseline и т.п.
- Мини‑статистика: % значений, попадающих в сетку.

## 5. Алгоритмы (уточнение к оригинальному ТЗ)

### 5.1 Индикатор ритма

Берём все активные параметры в px: `heading-size`, `section-gap`, `container-width`, а также вычисленную `line-height-px = round(heading-size × line-height)` и список вертикальных отступов пресета (`--gap-s`, `--gap-m`, `--gap-l`).

Для каждого значения `v` и выбранного модуля `m` считаем «штраф» `p = min(v mod m, m − v mod m)`. Доля «чистых» значений `clean = (кол-во v с p === 0) / total`.

- `clean === 1` → 🟢 «Ритм соблюдён».
- `0.6 ≤ clean < 1` → 🟡 «Есть отклонения», показываем конкретный список.
- `clean < 0.6` → 🔴 «Ритм нарушен».

Также `line-height × heading-size` должно быть кратно `m` — если нет, добавляем отдельное предупреждение «baseline плывёт».

### 5.2 Тепловая карта (F‑pattern)

Не симулируем eye‑tracking, а рисуем детерминированную «карту внимания» на основе DOM:

1. Для каждого «значимого» элемента (по CSS‑классу `.eye-target`) берём его BoundingClientRect относительно сцены.
2. Присваиваем вес по типу:
   - `h1` → 1.0, `h2` → 0.8, `CTA` → 0.95, `изображение` → 0.9, `цитата` → 0.7, `абзац` → 0.4, `тег/мелочь` → 0.2.
3. Усиливаем вес по F‑паттерну: множитель `1 − (y / sceneHeight) × 0.4`, и для первой «перекладины» F ( `y < 0.25·h` и `x < 0.6·w`) `+0.2`.
4. Рисуем на Canvas радиальные градиенты (красный центр → прозрачный) с радиусом `max(w,h) / 2` и альфой = вес.
5. `globalCompositeOperation = 'lighter'` для суммирования.
6. Синий — отсутствие внимания (фон Canvas слегка синит).

Карта пересчитывается на `resize`, смене пресета и изменении любого параметра.

### 5.3 Режим «До/После»

«После» — текущее состояние. «До» — то же содержимое с ломаной типографикой: случайные отступы (не кратные модулю), line-height 1.1, ширина контейнера 100%, заголовки без иерархии. Переключение через слайдер (вертикальная линия, движется мышью/тачем).

### 5.4 Дальтонизм

Применяется к сцене через CSS `filter: url(#protanopia)` и т.п. Матрицы `<feColorMatrix>` — стандартные Brettel/Viénot (инлайн в `index.html`).

## 6. Контент пресетов

Полностью по оригинальному ТЗ (лендинг системы полива, статья о модульных сетках, карточка Nike Air Max). Плюс к этому: у каждого значимого элемента есть `data-note` — заметка дизайнера (2–3 строки). Пример:

- Заголовок лендинга → «Заголовок занимает 1 строку на desktop, вес 700. Размер = 6× модуль, чтобы выделяться из иерархии, но оставаться на baseline.»
- CTA → «Высота кнопки = 6× модуль. Горизонтальный padding = 3× модуль. Это даёт минимальную тач‑цель 44 px при модуле 8.»

## 7. Адаптивность и доступность

- Минимальная поддерживаемая ширина 360 px. На мобильных панели сворачиваются в «ящики» (накладываются сверху по кнопке).
- Ползунки управляются клавиатурой (`←`/`→`, шаги как в числовом значении).
- Все контролы имеют `aria-label` и видимый фокус.
- Цветовой контраст бейджа ритма ≥ AA на обеих темах.
- `prefers-reduced-motion`: отключает «вырастание» сетки и микро‑анимации.

---

## 8. Критерии готовности (Definition of Done)

1. Все 3 пресета отображаются без горизонтального скролла на 360–1440 px.
2. Изменение любого параметра обновляет сцену, тепловую карту и индикатор ритма без видимой задержки (< 50 ms).
3. Индикатор ритма проходит юнит‑проверку на 5 контрольных комбинациях (описаны в `scripts/rhythm.js`).
4. PNG‑экспорт сохраняет ровно видимую область сцены (без панелей).
5. Все режимы дальтонизма визуально различимы.
6. Lighthouse: Performance ≥ 90, Accessibility ≥ 95 (desktop).
7. Сайт задеплоен как статика, ссылка приложена к PR.
